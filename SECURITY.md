# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please follow these steps to report it:

1. **Confidentiality:** Please do not disclose the vulnerability publicly until it has been addressed.
2. **Contact Information:** Send an email to [your-email@example.com] to report the issue.
3. **Details:** Include the following information in your report:
   - A description of the vulnerability.
   - Steps to reproduce the issue.
   - Any additional context that may help us understand the impact.

## Security Practices

For general security, we recommend the following best practices while contributing to this project:

- Keep your dependencies updated.
- Review all pull requests for potential security issues.
- Use environment variables for sensitive information.
- Regularly audit your code for vulnerabilities.

Thank you for helping to keep our project secure!
